     План: Совместные задачи между друзьями

     Контекст

     Пользователь хочет фичу: один PRO-пользователь может создать «совместную задачу» с другом, назначить подзадачи исполнителям (себе или другу),
     предложить другу принять/отклонить. После принятия задача появляется на Dashboard обоих, каждый может отмечать свои подзадачи выполненными.

     ---
     Архитектура

     Хранилище: PostgreSQL в users-service (не трогаем tasks-service)
     Видимость: в раздел Друзья (третья вкладка) + секция на Dashboard
     Файлы фронта: shared_tasks.js (новый) + правки в friends.js, tasks.js, index.html

     ---
     Backend — users-service

     1. Миграция 0005_create_shared_tasks.up.sql

     CREATE TABLE IF NOT EXISTS shared_tasks (
         id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
         proposer_id  UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
         addressee_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
         title        TEXT NOT NULL,
         description  TEXT NOT NULL DEFAULT '',
         status       VARCHAR(20) NOT NULL DEFAULT 'pending',
         created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
         updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
     );
     CREATE TABLE IF NOT EXISTS shared_subtasks (
         id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
         shared_task_id UUID NOT NULL REFERENCES shared_tasks(id) ON DELETE CASCADE,
         title          TEXT NOT NULL,
         assignee_id    UUID NOT NULL REFERENCES users(user_id),
         is_done        BOOLEAN NOT NULL DEFAULT FALSE,
         order_num      INT NOT NULL DEFAULT 0,
         created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
     );
     CREATE INDEX IF NOT EXISTS idx_st_proposer   ON shared_tasks(proposer_id);
     CREATE INDEX IF NOT EXISTS idx_st_addressee  ON shared_tasks(addressee_id);
     CREATE INDEX IF NOT EXISTS idx_sst_task      ON shared_subtasks(shared_task_id);

     3. Новые модели (model.go)

     type SharedTask struct {
         ID            string          `json:"id"`
         ProposerID    string          `json:"proposer_id"`
         AddresseeID   string          `json:"addressee_id"`
         Title         string          `json:"title"`
         Description   string          `json:"description"`
         Status        string          `json:"status"` // pending|accepted|rejected
         CreatedAt     time.Time       `json:"created_at"`
         Subtasks      []SharedSubtask `json:"subtasks"`
         ProposerName  string          `json:"proposer_name,omitempty"`
         AddresseeName string          `json:"addressee_name,omitempty"`
     }
     type SharedSubtask struct {
         ID           string `json:"id"`
         SharedTaskID string `json:"shared_task_id"`
         Title        string `json:"title"`
         AssigneeID   string `json:"assignee_id"`
         AssigneeName string `json:"assignee_name,omitempty"`
         IsDone       bool   `json:"is_done"`
         OrderNum     int    `json:"order_num"`
     }
     type ProposeSharedTaskDTO struct {
         AddresseeID string       `json:"addressee_id"`
         Title       string       `json:"title"`
         Description string       `json:"description"`
         Subtasks    []SubtaskDTO `json:"subtasks"`
     }
     type SubtaskDTO struct {
         Title      string `json:"title"`
         AssigneeID string `json:"assignee_id"`
     }

     4. Интерфейс репозитория (storagePostgres/storage.go)

     Добавить к PostgresRepository:
     ProposeSharedTask(ctx, proposerID, addresseeID, title, desc string, subtasks []model.SubtaskDTO) (string, error)
     GetSharedTasks(ctx, userID string) ([]model.SharedTask, error)
     GetSharedTaskDetail(ctx, taskID, userID string) (*model.SharedTask, error)
     RespondSharedTask(ctx, taskID, userID, status string) error
     ToggleSubtaskDone(ctx, subtaskID, userID string) (bool, error) // возвращает новый статус

     5. Реализация (db/postgresql/postgresql.go)

     ProposeSharedTask: транзакция INSERT shared_tasks + INSERT каждого subtask
     GetSharedTasks: JOIN users для имён, WHERE (proposer_id=$1 OR addressee_id=$1), ORDER BY created_at DESC, + второй запрос для subtasks
     GetSharedTaskDetail: то же + проверка доступа (proposer или addressee = userID)
     RespondSharedTask: UPDATE SET status=$2 WHERE id=$1 AND addressee_id=$3 AND status='pending'
     ToggleSubtaskDone: UPDATE SET is_done = NOT is_done WHERE id=$1 AND assignee_id=$2 RETURNING is_done

     6. Сервис (service.go)

     Проверки:
     - В ProposeSharedTask: оба пользователя должны быть друзьями (AreFriends)
     - AssigneeID каждой подзадачи должен быть proposer или addressee (иначе 400)

     7. Роуты и хендлеры (hander.go)

     Новые URL-константы:
     sharedTasksURL        = "/users/:uuid/shared-tasks"
     sharedTaskDetailURL   = "/users/:uuid/shared-tasks/:stid"
     sharedTaskAcceptURL   = "/users/:uuid/shared-tasks/:stid/accept"
     sharedTaskRejectURL   = "/users/:uuid/shared-tasks/:stid/reject"
     sharedSubtaskDoneURL  = "/users/:uuid/shared-tasks/:stid/subtasks/:sid"
     Регистрация:
     router.HandlerFunc(GET,   sharedTasksURL,       mw(h.GetSharedTasks))
     router.HandlerFunc(POST,  sharedTasksURL,       mw(h.ProposeSharedTask))
     router.HandlerFunc(GET,   sharedTaskDetailURL,  mw(h.GetSharedTaskDetail))
     router.HandlerFunc(POST,  sharedTaskAcceptURL,  mw(h.AcceptSharedTask))
     router.HandlerFunc(POST,  sharedTaskRejectURL,  mw(h.RejectSharedTask))
     router.HandlerFunc(PATCH, sharedSubtaskDoneURL, mw(h.ToggleSubtaskDone))
     ⚠️ Конфликтов нет: :stid имеет статические дочерние accept, reject, subtasks — паттерн аналогичен существующему friends/:fid/accept.
